# Attendance module ŌĆö database schema

Source: `server/attendance/attendance.js` (SQLite). Postgres DDL:
`docs/db/modules/attendance.sql`.

This module owns two tables. All foreign keys are **explicit in the SQLite source**
(declared with `REFERENCES`), so none are inferred.

---

## Table: `attendance_vouchers`

Header record for an attendance voucher belonging to a company.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `attendance_voucher_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | ŌĆö | FK -> `companies.company_id` (ON DELETE CASCADE). |
| `voucher_number` | `TEXT` | yes | ŌĆö | Auto-generated `ATT-NNNNN` if not supplied at create. |
| `date` | `DATE` | no | ŌĆö | Voucher date. SQLite `TEXT NOT NULL` (date-only) -> `DATE`. |
| `narration` | `TEXT` | yes | ŌĆö | Free-text note. |
| `created_at` | `TIMESTAMPTZ` | yes | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; app stores ISO-8601 strings. |
| `updated_at` | `TIMESTAMPTZ` | yes | `now()` | Same default; **never updated** ŌĆö no update path in this module. |

### Relationships
- `company_id` -> `companies(company_id)` ŌĆö explicit, `ON DELETE CASCADE`.
- Referenced by `attendance_voucher_entries.attendance_voucher_id` (cascade delete).

## Table: `attendance_voucher_entries`

Per-employee line under a voucher.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `entry_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `attendance_voucher_id` | `BIGINT` | no | ŌĆö | FK -> `attendance_vouchers.attendance_voucher_id` (ON DELETE CASCADE). |
| `employee_id` | `BIGINT` | yes | ŌĆö | FK -> `employees.employee_id`. Nullable (no NOT NULL in source). |
| `attendance_type_id` | `BIGINT` | yes | ŌĆö | FK -> `attendance_types.attendance_type_id`. Nullable. |
| `value` | `NUMERIC(18,4)` | yes | `0` | Quantity/value. SQLite `REAL DEFAULT 0`. Mapped to NUMERIC, **never floating point**, for accounting precision. |

### Relationships
- `attendance_voucher_id` -> `attendance_vouchers(attendance_voucher_id)` ŌĆö explicit, `ON DELETE CASCADE`.
- `employee_id` -> `employees(employee_id)` ŌĆö explicit (no ON DELETE action in source).
- `attendance_type_id` -> `attendance_types(attendance_type_id)` ŌĆö explicit (no ON DELETE action in source).

## Indexes (added in DDL)
- `idx_attendance_vouchers_company_id` on `attendance_vouchers(company_id)`
- `idx_attendance_voucher_entries_voucher_id` on `attendance_voucher_entries(attendance_voucher_id)`
- `idx_attendance_voucher_entries_employee_id` on `attendance_voucher_entries(employee_id)`
- `idx_attendance_voucher_entries_attendance_type_id` on `attendance_voucher_entries(attendance_type_id)`

## Notes
- No UNIQUE constraints exist in the source. `voucher_number` is **not** unique at
  the DB level; uniqueness within a company is only enforced procedurally by the
  `MAX(...)+1` generator in `attendanceService.js`.
- No boolean/flag columns and no currency-amount columns in this module; `value`
  is the only numeric (treated as a quantity -> `NUMERIC(18,4)`).
