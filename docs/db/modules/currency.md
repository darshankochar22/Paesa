# Currency Module — Database Tables

Source: `server/currency/currency.js` (SQLite). Target: PostgreSQL.
This module owns one table: `currencies`.

## Table: `currencies`

Stores the currencies configured per company. The predefined Indian Rupee (INR)
base currency is seeded on company creation via `seedDefaultCurrency()`.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `currency_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies.company_id` (explicit in source, `ON DELETE CASCADE`). |
| `name` | `TEXT` | no | — | Display name. |
| `formal_name` | `TEXT` | yes | — | On create, defaults to `name` if omitted. |
| `iso_code` | `TEXT` | no | — | ISO currency code (e.g. INR). Uniqueness enforced case-insensitively among active rows. |
| `symbol` | `TEXT` | yes | — | Currency symbol (e.g. ₹). |
| `decimal_places` | `INTEGER` | no | `2` | Number of decimal places. |
| `decimal_symbol` | `TEXT` | no | `'.'` | Decimal separator. |
| `decimal_places_in_words` | `TEXT` | yes | — | e.g. "Paise". |
| `suffix_symbol_to_amount` | `BOOLEAN` | no | `false` | SQLite `INTEGER 0/1` -> BOOLEAN (0=false, 1=true). |
| `show_amount_in_millions` | `BOOLEAN` | no | `false` | SQLite `INTEGER 0/1` -> BOOLEAN. |
| `word_representing_amount_after_decimal` | `TEXT` | yes | — | e.g. "Paise". |
| `add_space_between_amount_and_symbol` | `BOOLEAN` | no | `false` | SQLite `INTEGER 0/1` -> BOOLEAN. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER 0/1` -> BOOLEAN. Soft-delete sets this `false`. |
| `is_default` | `BOOLEAN` | no | `false` | SQLite `INTEGER 0/1` -> BOOLEAN. Company default currency. |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER 0/1` -> BOOLEAN. Marks the seeded base currency (cannot edit/delete). |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`, stored as ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`. Refreshed on update. |

### Type-mapping callouts
- **Boolean flags**: six `INTEGER 0/1` columns become `BOOLEAN` in Postgres (0 -> false, 1 -> true). The service coerces inputs with `value ? 1 : 0`.
- **Datetimes**: `created_at` / `updated_at` are ISO-8601 strings in SQLite; mapped to `TIMESTAMPTZ` with `now()` defaults.
- **No money/quantity columns** exist in this table, so no `NUMERIC` precision decisions were required.

### Constraints & indexes
- `UNIQUE` (partial): `(company_id, LOWER(iso_code)) WHERE is_active = true` — mirrors the service's case-insensitive duplicate-active check. Not present in the SQLite source; recommended for Postgres.
- Index `idx_currencies_company_id` on the FK column.

## Relationships

- `currencies.company_id` -> `companies.company_id` — **explicit FK** in the SQLite source (`REFERENCES companies(company_id) ON DELETE CASCADE`). Deleting a company cascades to its currencies. `companies` is owned by the company module; the FK is emitted as an `ALTER TABLE` at the bottom of `currency.sql` to avoid cross-module load-ordering issues.
