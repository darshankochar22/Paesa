# dayBookReport — database schema

Source: `server/dayBookReport/dayBookReport.js` (SQLite). Postgres DDL in
`docs/db/modules/dayBookReport.sql`.

Three tables: `day_book_reports` (header) → `day_book_entries` → `day_book_entry_lines`.

General mapping notes:
- `INTEGER PRIMARY KEY AUTOINCREMENT` → `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- 0/1 INTEGER flags → `BOOLEAN` (migrate `<> 0` → true). Defaults preserved.
- Money REAL columns → `NUMERIC(18,2)` — **never** floating point.
- `datetime('now')` TEXT defaults → `TIMESTAMPTZ DEFAULT now()` (app stores ISO strings).
- date-only TEXT (`date_from`, `date_to`, `voucher_date`) → `DATE`.

## Cross-module references
`companies`, `vouchers`, and `ledgers` are owned by other modules. FK constraints are
emitted as `ALTER TABLE` statements at the bottom of the `.sql` file so load order does
not matter.
