# costCentre module — database schema

Source: `server/costCentre/costCentre.js`
PostgreSQL DDL: `docs/db/modules/costCentre.sql`

This module owns a single table: **`cost_centres`** (a self-referential
hierarchy of cost centres scoped to a company).

## Table: `cost_centres`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `cc_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies.company_id` (explicit, `ON DELETE CASCADE`). |
| `name` | `TEXT` | no | — | Cost centre name. Case-insensitive uniqueness per active company enforced in app code. |
| `alias` | `TEXT` | yes | — | Optional short alias. |
| `parent_id` | `BIGINT` | yes | — | Self-FK -> `cost_centres.cc_id` (explicit). NULL = top-level. |
| `category` | `TEXT` | no | `'Primary'` | Derived server-side: `'Secondary'` when `parent_id` set, else `'Primary'`. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER` 0/1 (DEFAULT 1). Soft-delete sets `false`. 1->true, 0->false. |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER` 0/1 (DEFAULT 0). Seeded/system rows. 1->true, 0->false. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT` ISO-8601; default `datetime('now')`. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT` ISO-8601; default `datetime('now')`. Stamped on update. |

Notes:
- No money or quantity columns in this table, so the NUMERIC(18,2)/(18,4)
  money rule does not apply here.
- The 0/1 -> BOOLEAN conversion applies to `is_active` and `is_predefined`
  (0 -> false, 1 -> true).

## Relationships

- **`company_id` -> `companies.company_id`** — explicit in SQLite source,
  `ON DELETE CASCADE`. Deleting a company removes its cost centres.
- **`parent_id` -> `cost_centres.cc_id`** — explicit, self-referential
  parent/child hierarchy. No cascade specified; the app blocks deleting a
  centre that still has active children (service layer), and uses a soft delete
  (`is_active = false`) rather than a hard delete.

Both FKs are emitted as separate `ALTER TABLE ... ADD CONSTRAINT` statements at
the bottom of `costCentre.sql`. Both are **explicit** in the SQLite source (not
inferred). Supporting indexes exist on `company_id` and `parent_id`.
