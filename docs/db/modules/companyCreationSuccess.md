# companyCreationSuccess — Database Schema

Source: `server/companyCreationSuccess/companyCreationSuccess.js`
Postgres DDL: `docs/db/modules/companyCreationSuccess.sql`

This module defines a single table that stores per-company onboarding /
"created successfully" UI state and feature-setup flags.

## Table: `company_creation_success`

One row per company (seeded at company creation). All flag columns are
SQLite INTEGER 0/1 booleans translated to Postgres `BOOLEAN`
(0 -> `false`, 1 -> `true`).

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies(company_id)`, ON DELETE CASCADE. SQLite `NOT NULL`. |
| `created_successfully` | `BOOLEAN` | no | `TRUE` | SQLite INTEGER 0/1, DEFAULT 1. Company created OK. |
| `success_screen_shown` | `BOOLEAN` | no | `FALSE` | SQLite INTEGER 0/1, DEFAULT 0. Success screen displayed. |
| `show_more_features` | `BOOLEAN` | no | `FALSE` | SQLite INTEGER 0/1, DEFAULT 0. UI flag. |
| `show_all_features` | `BOOLEAN` | no | `FALSE` | SQLite INTEGER 0/1, DEFAULT 0. UI flag. |
| `default_features_loaded` | `BOOLEAN` | no | `TRUE` | SQLite INTEGER 0/1, DEFAULT 1. Default features loaded. |
| `feature_setup_completed` | `BOOLEAN` | no | `FALSE` | SQLite INTEGER 0/1, DEFAULT 0. Feature setup finished. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite TEXT `datetime('now')`; app stores ISO-8601 strings. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite TEXT `datetime('now')`; bumped on every update. |

### Constraints & Indexes

- `uq_company_creation_success_company_id` — UNIQUE on `company_id`. **Inferred**, not in SQLite source; the service treats `company_id` as the single lookup/update key (one row per company).
- `idx_company_creation_success_company_id` — index on the FK column.

### Relationships

- `company_id` → `companies(company_id)` — **explicit** FK in the SQLite source (`REFERENCES companies(company_id) ON DELETE CASCADE`). Deleting a company cascades and removes its creation-success record.

### Notes

- No money/quantity columns exist in this module, so no NUMERIC precision concerns apply.
- There is no monetary or floating-point data here; all non-key columns are boolean flags or timestamps.
