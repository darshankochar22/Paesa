# Voucher Module — Database Contract

Source of truth: `server/voucher/voucher.js` (SQLite `CREATE TABLE` init).
Postgres DDL: [`voucher.sql`](./voucher.sql).

13 tables. Conventions:
- SQLite `INTEGER PRIMARY KEY AUTOINCREMENT` → Postgres `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- SQLite `INTEGER 0/1` flag columns (`is_*`) → Postgres `BOOLEAN` (0 → false, 1 → true).
- ISO-8601 datetime TEXT (`created_at`, `updated_at`) → `TIMESTAMPTZ DEFAULT now()` (app stores ISO strings).
- date-only TEXT (`date`, `*_date`) → `DATE`.
- **Money columns map to `NUMERIC(18,2)` and quantity/rate to `NUMERIC(18,4)` — never floating point**, even though SQLite stores them as `REAL`. This is mandatory for an accounting ledger.
- FK constraints live in `voucher.sql` as grouped `ALTER TABLE` statements; relationships are listed per table below.
