## Table: `tds_nature_of_payment`

### Type-mapping callouts

- **Money / rates are NUMERIC, never floating point.** `threshold_limit` → `NUMERIC(18,2)` (currency); `rate_individual_with_pan` and `rate_other_with_pan` → `NUMERIC(18,4)` (percentage rates needing extra precision).
- **Boolean conversion.** `is_zero_rated`, `is_predefined`, `is_active` are SQLite `INTEGER` 0/1 → Postgres `BOOLEAN`; migrate `0 → false`, `1 → true`.
- **Datetime conversion.** `created_at` / `updated_at` store ISO-8601 strings in SQLite → `TIMESTAMPTZ`; cast string values on migration. SQLite `datetime('now')` default → Postgres `now()`.

### Relationships

- `company_id` → `companies(company_id)` — **EXPLICIT in source** (`REFERENCES companies(company_id) ON DELETE CASCADE`). Deleting a company cascades to delete its TDS nature-of-payment records. Indexed via `idx_tds_nature_of_payment_company_id`.

No other `*_id` columns exist on this table, so no additional FKs are inferred.

### Constraints & indexes

- Primary key: `tds_id`.
- No `UNIQUE` constraint in source — uniqueness of `(company_id, LOWER(name))` among active rows is enforced only in application logic (`create`), not at the DB level.
- Index `idx_tds_nature_of_payment_company_id` on `(company_id)`.
- Composite index `idx_tds_nature_of_payment_company_active` on `(company_id, is_active)` to support the per-company active lookups (`getAll`, create dup-check).
