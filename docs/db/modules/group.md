# Group Module — Database Reference

Source: `server/group/group.js` (SQLite schema init). Postgres DDL: [`group.sql`](./group.sql).

This module owns a single table: **`groups`** — the chart-of-accounts hierarchy
(Tally-style primary / predefined / user account groups).

### Money / numeric handling

- `gst_rate`, `cgst_rate`, `sgst_rate`, `igst_rate` are GST tax **percentages**, not
  currency. They were SQLite `REAL`. They are mapped to `NUMERIC(18,4)` so tax computations
  stay exact — **never** map these to `DOUBLE PRECISION` / floating point.
- There are no currency-amount columns in this table.

### Boolean conversion

`is_primary`, `is_predefined`, `set_alter_tds_details`, `set_alter_tcs_details`,
`set_alter_other_statutory_details`, `behaves_like_subledger`, `show_net_debit_credit`,
`used_for_calculation`, `is_active` were SQLite `INTEGER` flags storing `0`/`1`. In Postgres
they are `BOOLEAN`; migrate with `(col <> 0)` (0 -> false, 1 -> true).

## Relationships

- **groups.company_id -> companies.company_id** — *explicit* FK in the SQLite source
  (`REFERENCES companies(company_id) ON DELETE CASCADE`). Each group belongs to one company.
- **groups.parent_group_id -> groups.group_id** — *explicit* self-referential FK in the
  SQLite source. Models the parent/child hierarchy consumed by `getTree`. Postgres DDL uses
  `ON DELETE RESTRICT` to mirror the service guard that blocks deleting a group with active
  subgroups.

### Constraints / indexes added in Postgres

- Unique (active) group name per company, case-insensitive:
  `uq_groups_company_active_name (company_id, LOWER(name)) WHERE is_active = true` — enforces
  the service's pre-insert duplicate check.
- Indexes on `company_id`, `parent_group_id`, and `(company_id, is_active)` for the common
  getAll / getTree lookups.
