# Unit Module — Database Schema

Source: `server/unit/unit.js` (SQLite). Target dialect documented: **PostgreSQL**.
DDL: `docs/db/modules/unit.sql`.

This module owns one table: **`units`**. It supports both Simple units and Compound
units (a compound unit self-references two simple units with a conversion factor).

## Table: `units`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `unit_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Owning company. FK -> `companies.company_id` (ON DELETE CASCADE). |
| `name` | `TEXT` | no | — | Unit display name. |
| `symbol` | `TEXT` | no | — | Short symbol (e.g. `Kg`). Uniqueness enforced in app logic per company (case-insensitive, active only), not by a DB constraint. |
| `formal_name` | `TEXT` | yes | — | Defaults to `name` in service when omitted. |
| `decimal_places` | `INTEGER` | no | `0` | SQLite `DEFAULT 0`. |
| `unit_quantity_code` | `TEXT` | yes | — | Optional UQC / quantity code. |
| `unit_type` | `TEXT` | no | `'Simple'` | `'Simple'` or `'Compound'`. SQLite `DEFAULT 'Simple'`. |
| `is_simple` | `BOOLEAN` | no | `TRUE` | SQLite `INTEGER` 0/1 (`DEFAULT 1`). 0->false, 1->true. |
| `is_active` | `BOOLEAN` | no | `TRUE` | SQLite `INTEGER` 0/1 (`DEFAULT 1`). Soft-delete flag (delete sets false). |
| `is_predefined` | `BOOLEAN` | no | `FALSE` | SQLite `INTEGER` 0/1 (`DEFAULT 0`). Seeded defaults = true; cannot edit/delete. |
| `first_unit_id` | `BIGINT` | yes | — | Compound only. Self-FK -> `units.unit_id`. Added via migration ALTER. |
| `second_unit_id` | `BIGINT` | yes | — | Compound only. Self-FK -> `units.unit_id`. Added via migration ALTER. |
| `conversion_factor` | `NUMERIC(18,4)` | yes | `1` | Compound ratio (`first_unit = factor x second_unit`). **NUMERIC, never floating point.** |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; app stores ISO-8601 strings. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; bumped on update via `datetime('now')`. |

### Boolean conversion

`is_simple`, `is_active`, and `is_predefined` are stored in SQLite as `INTEGER` `0`/`1`.
In Postgres they become `BOOLEAN`: `0 -> false`, `1 -> true`. Service defaults (1, 1, 0)
map to (`TRUE`, `TRUE`, `FALSE`).

### Money / quantity precision

`conversion_factor` is a quantity ratio and is mapped to `NUMERIC(18,4)`, never a
floating-point type, to preserve exact conversion arithmetic. This is the only
numeric-ratio/money-like column in this module.

## Relationships

- **`units.company_id` -> `companies.company_id`** — *explicit* FK in SQLite source
  (`REFERENCES companies(company_id) ON DELETE CASCADE`). Deleting a company cascades
  its units.
- **`units.first_unit_id` -> `units.unit_id`** — *explicit* self-reference (compound
  unit's first/base unit). Nullable; only set for compound units.
- **`units.second_unit_id` -> `units.unit_id`** — *explicit* self-reference (compound
  unit's second/related unit). Nullable; only set for compound units.

Indexes added in Postgres: `idx_units_company_id`, `idx_units_first_unit_id`,
`idx_units_second_unit_id`, and a composite `idx_units_company_active`
(`company_id, is_active`) for the common "active units per company" query.

## Migration note

`unit.js` runs idempotent `ALTER TABLE units ADD COLUMN ...` for `first_unit_id`,
`second_unit_id`, and `conversion_factor` (wrapped in try/catch) to upgrade older
databases that predate the compound-unit feature. The consolidated `CREATE TABLE` in
`unit.sql` already includes these columns, so no separate ALTER is needed for fresh
Postgres installs.
