# Godown Module — Database Schema

Source: `server/godown/godown.js` (SQLite init). Target translation: PostgreSQL
(see `docs/db/modules/godown.sql`).

This module owns a single table: **`godowns`**.

## Table: `godowns`

Storage / inventory locations, hierarchical (self-referential parent), scoped per company.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `godown_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies.company_id` (explicit, `ON DELETE CASCADE`). |
| `name` | `TEXT` | no | — | Godown name. Active duplicates per company rejected in service. |
| `alias` | `TEXT` | yes | — | Optional alias. |
| `parent_godown_id` | `BIGINT` | yes | — | Self-FK -> `godowns.godown_id` (explicit). `NULL` = root godown. |
| `address` | `TEXT` | yes | — | |
| `city` | `TEXT` | yes | — | |
| `state` | `TEXT` | yes | — | |
| `pincode` | `TEXT` | yes | — | Stored as text (preserves leading zeros). |
| `is_primary` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0->false / 1->true. Set to `1` on create when no parent. |
| `is_main_location` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0->false / 1->true. |
| `allow_storage_of_materials` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1`. 0->false / 1->true. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1`. 0->false / 1->true. Soft-delete flag; delete sets to `false`. |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0->false / 1->true. `true` blocks edit/delete (Main Location). |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; refreshed on update. |

No money/quantity columns in this table, so no `NUMERIC` precision concerns apply here.

### Indexes
- `idx_godowns_company_id` on (`company_id`)
- `idx_godowns_parent_godown_id` on (`parent_godown_id`)
- `idx_godowns_company_active` on (`company_id`, `is_active`) — supports active-name lookups.

> Note: there is **no** DB-level UNIQUE constraint on the name. Uniqueness of an
> active godown name per company is enforced only in the service layer
> (`LOWER(name)` comparison among `is_active = 1` rows). Consider a partial unique
> index `UNIQUE (company_id, lower(name)) WHERE is_active` if enforcing at the DB.

## Relationships

- **`godowns.company_id` -> `companies.company_id`** (explicit in source,
  `ON DELETE CASCADE`). Each godown belongs to exactly one company.
- **`godowns.parent_godown_id` -> `godowns.godown_id`** (explicit in source,
  self-referential). A godown may have a parent godown; roots have `NULL`. The
  `getTree` operation builds the hierarchy from this column.
