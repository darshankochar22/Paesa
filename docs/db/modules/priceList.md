# priceList Module — Database Schema (PostgreSQL contract)

Source of truth: `server/priceList/priceList.js` (SQLite `CREATE TABLE` init).
This document is the Postgres translation. DDL lives in
`docs/db/modules/priceList.sql`.

Two tables: `price_lists` (headers) and `price_list_lines` (child rate/slab
lines). Both carry SQLite `INTEGER PRIMARY KEY AUTOINCREMENT` keys translated to
identity columns, and `created_at` / `updated_at` ISO-string timestamps
translated to `TIMESTAMPTZ DEFAULT now()`.

> **Money / quantity precision:** all numeric accounting columns
> (`qty_from`, `qty_less_than`, `rate`, `disc_percent`) are mapped to
> `NUMERIC(18,4)` — never floating point — to avoid rounding error in an
> accounting system.

---

## Table: `price_lists`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `price_list_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. From SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies.company_id` (explicit in source, `ON DELETE CASCADE`). |
| `stock_group` | `TEXT` | no | `'All Items'` | Stock group filter. |
| `price_level` | `TEXT` | no | — | Price level / tier name. |
| `applicable_from` | `DATE` | no | — | Effective date. SQLite stored it as TEXT; mapped to DATE. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1` used as 0/1 boolean (0=false, 1=true). Soft-delete flag. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `datetime('now')` ISO string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `datetime('now')` ISO string; bumped on update/soft-delete. |

**Relationships**
- `company_id` -> `companies.company_id` — **explicit** FK in SQLite source,
  `ON DELETE CASCADE`. Index: `idx_price_lists_company_id`.
- Referenced by `price_list_lines.price_list_id` (children).

## Table: `price_list_lines`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `line_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. From SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |

## Nullability / default notes

- The SQLite source marks several columns only with `DEFAULT` (e.g. `is_active`,
  `qty_*`, `rate`, `disc_percent`, `sort_order`, `created_at`, `updated_at`).
  Per the mapping rules, columns with a `DEFAULT` are emitted `NOT NULL` in
  Postgres since a value is always materialised.
- `item_id` has no `NOT NULL` and is the only intentionally nullable column;
  its FK uses `ON DELETE SET NULL`.
- `applicable_from` is `NOT NULL` in source and stored as a date string by the
  service; mapped to `DATE`.

## Behavioural notes (from the service layer)

- **Soft delete:** `priceList:delete` sets `is_active = false` and bumps
  `updated_at`; rows are not physically removed. `getAll`/`getById` filter
  `is_active = true`.
- **Line replace on update:** `priceList:update` deletes all
  `price_list_lines` for the header and re-inserts them, reassigning
  `sort_order` from the new array order.
