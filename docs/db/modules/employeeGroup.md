### Constraints & indexes
- Primary key: `employee_group_id`.
- Unique (partial): `(company_id, LOWER(name))` where `is_active = TRUE` ‚Äî formalizes the service-level duplicate-name check.
- Indexes: `company_id`, `employee_category_id`, `parent_group_id`.

### No money/quantity columns
This table has no currency or quantity fields, so no `NUMERIC(18,2)`/`NUMERIC(18,4)` mappings apply.

## Relationships

- **employee_groups.company_id -> companies.company_id** ‚Äî explicit FK in source, `ON DELETE CASCADE`. Each group belongs to exactly one company.
- **employee_groups.employee_category_id -> employee_categories.employee_category_id** ‚Äî explicit FK in source (nullable). Optional category assignment.
- **employee_groups.parent_group_id -> employee_groups.employee_group_id** ‚Äî explicit self-FK in source (nullable). Builds the group hierarchy used by `getTree`.
- **employees.employee_group_id -> employee_groups.employee_group_id** ‚Äî inferred (referenced by the `delete` guard `SELECT * FROM employees WHERE employee_group_id = ?`). Owned by the `employees` table/module, so the FK constraint belongs in that module's DDL, not here.
