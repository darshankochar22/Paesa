# payrollUnit module — database schema

Source: `server/payrollUnit/payrollUnit.js`
Postgres DDL: `payrollUnit.sql`

One table: **`payroll_units`** — units of measure used in payroll, scoped per
company. Includes seeded "predefined" units (Days/Hours/Minutes/Months/Pieces)
that cannot be edited or deleted.

## Table: `payroll_units`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `payroll_unit_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Owning company. FK -> `companies.company_id` (**explicit** in source, `ON DELETE CASCADE`). |
| `name` | `TEXT` | no | — | Unit name (e.g. "Hours"). |
| `symbol` | `TEXT` | yes | — | Short symbol (e.g. "Hrs"). |
| `formal_name` | `TEXT` | yes | — | Formal/long name. (Added via idempotent `ALTER TABLE ... ADD COLUMN`.) |
| `unit_type` | `TEXT` | no | `'Simple'` | Unit type. SQLite had a DEFAULT, so NOT NULL added. |
| `decimal_places` | `INTEGER` | no | `0` | Decimal precision for quantities. SQLite had a DEFAULT. |
| `first_unit` | `TEXT` | yes | — | First unit (compound units). |
| `conversion` | `NUMERIC(18,4)` | yes | — | Conversion factor between first/second unit. SQLite `REAL`; mapped to NUMERIC (quantity/rate) — **never floating point**. |
| `second_unit` | `TEXT` | yes | — | Second unit (compound units). |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER` 0/1 used as boolean (0->false, 1->true). 0 = soft-deleted. |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER` 0/1 used as boolean. true = system-seeded, immutable. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string; refreshed on update. |

### Constraints & indexes
- **PK**: `payroll_unit_id`.
- **Index**: `idx_payroll_units_company_id` on `company_id` (FK column).
- **Unique (recommended)**: `uq_payroll_units_company_name_active` on
  `(company_id, LOWER(name)) WHERE is_active = true` — mirrors the app's
  case-insensitive duplicate guard among active units. Not present in the SQLite
  source; enforced by application logic there.

### Relationships
- `payroll_units.company_id` -> `companies.company_id`
  — **explicit FK in source** (`REFERENCES companies(company_id) ON DELETE CASCADE`).
  Deleting a company cascades to its payroll units.

### Money / numeric callout
- `conversion` is a unit-conversion factor (quantity/rate), mapped to
  `NUMERIC(18,4)`. There are **no currency/money columns** in this table.
- Quantities use `decimal_places` to indicate display precision but are not
  stored as money. No floating-point types are used for numeric business values.
