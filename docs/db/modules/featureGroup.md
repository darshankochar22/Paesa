# featureGroup module — database schema

Source of truth: `server/featureGroup/featureGroup.js` (SQLite init). Postgres DDL: `docs/db/modules/featureGroup.sql`.

This module owns one table: **`feature_groups`** — a small reference/lookup table of high-level feature categories, seeded at init time.

## Table: `feature_groups`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `feature_group_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` | no | identity | From SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `group_key` | `TEXT` | no | — | `NOT NULL UNIQUE`. Stable machine key, e.g. `accounts`, `inventory`, `gst`, `payroll`, `banking`, `online`. |
| `group_name` | `TEXT` | no | — | `NOT NULL`. Human-readable label. |
| `online_access` | `BOOLEAN` | no | `FALSE` | SQLite `INTEGER DEFAULT 0` used as a 0/1 boolean (0 -> false, 1 -> true). Made `NOT NULL` because source has a DEFAULT. |
| `display_order` | `INTEGER` | no | `0` | SQLite `INTEGER DEFAULT 0`. UI sort order. Made `NOT NULL` because source has a DEFAULT. |
| `is_active` | `BOOLEAN` | no | `TRUE` | SQLite `INTEGER DEFAULT 1` used as a 0/1 boolean (0 -> false, 1 -> true). `getAll` filters on `is_active = true`. Made `NOT NULL` because source has a DEFAULT. |

### Constraints & indexes
- `PRIMARY KEY (feature_group_id)`
- `UNIQUE (group_key)` — preserved from SQLite (`uq_feature_groups_group_key`).
- Index `idx_feature_groups_active_order (is_active, display_order)` — supports the `getAll` query (`WHERE is_active = 1 ORDER BY display_order ASC`).

### Seed data
`featureGroup.js` seeds six rows via `INSERT OR IGNORE`:

| group_key | group_name | online_access | display_order |
|---|---|---|---|
| accounts | Accounting Features | 0 | 1 |
| inventory | Inventory Features | 0 | 2 |
| gst | GST & Statutory Features | 0 | 3 |
| payroll | Payroll Features | 0 | 4 |
| banking | Banking Features | 0 | 5 |
| online | Online & Connected Features | 1 | 6 |

## Relationships

None. The `feature_groups` table has no `*_id` reference columns, so no foreign keys (explicit or inferred) are emitted. `feature_group_id` is this table's own primary key.

Note: conceptually a `feature_items`/`featureItem` module may reference `feature_group_id`, but any such FK belongs to that owning module's DDL, not this one.

## Type-mapping notes
- No money/quantity columns exist in this table, so no `NUMERIC` precision concerns apply here.
- The two 0/1 integer flags (`online_access`, `is_active`) are translated to `BOOLEAN`; consumers must convert SQLite `0/1` to Postgres `false/true`.
