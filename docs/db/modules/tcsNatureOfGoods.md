# DB Module: tcsNatureOfGoods

Source: `server/tcsNatureOfGoods/tcsNatureOfGoods.js`

One table: `tcs_nature_of_goods`. Stores TCS (Tax Collected at Source) Nature of Goods master records per company.

## Table: `tcs_nature_of_goods`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `tcs_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | FK -> `companies.company_id` (explicit in source). |
| `name` | `TEXT` | no | — | Record name; uniqueness enforced in app code (case-insensitive, among active rows), not by a DB constraint. |
| `section` | `TEXT` | yes | — | TCS section. |
| `payment_code` | `TEXT` | yes | — | Payment code. |
| `rate_individual_with_pan` | `NUMERIC(18,4)` | no | `0` | Rate value. Money/rate — never floating point. |
| `rate_individual_without_pan` | `NUMERIC(18,4)` | no | `0` | Rate value. Money/rate — never floating point. |

| `rate_other_without_pan` | `NUMERIC(18,4)` | no | `0` | Rate value. Money/rate — never floating point. |

| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite stores ISO-8601 string via `datetime('now')`; set on update. |

### NOT NULL / DEFAULT translation
All columns that carry a SQLite `DEFAULT` (every `rate_*`, the `is_*` flags, `tax_on_receipt_or_realization`, `threshold_level`, `created_at`, `updated_at`) are made `NOT NULL` with the corresponding Postgres default. `name` is `NOT NULL` in source. `company_id` is `NOT NULL` in source. `section` and `payment_code` have no default and remain nullable.

### Relationships
- `company_id` -> `companies(company_id)` — **explicit** FK in source SQLite DDL: `REFERENCES companies(company_id) ON DELETE CASCADE`. Preserved as an `ALTER TABLE` constraint with `ON DELETE CASCADE`. Indexed via `idx_tcs_nature_of_goods_company_id`.

### Constraints / indexes
- No `UNIQUE` constraint exists in the source schema; the "name already exists" check is performed in application code (`tcsNatureOfGoodsService.create`) over active rows of the same company. Not translated to a DB constraint to preserve behavior, though a partial unique index on `(company_id, lower(name)) WHERE is_active` could be added if DB-level enforcement is desired.
- Index added on the FK column `company_id`.
