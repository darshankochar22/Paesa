# financialYear module ‚Äî database reference

Source schema: `server/financialYear/financialYear.js` (SQLite).
Postgres translation: `docs/db/modules/financialYear.sql`.

This module owns one table: **`financial_years`**.

## Table: `financial_years`

A financial (accounting) year scoped to a company. Exactly one FY per company is expected to be active at a time (enforced in application code via `setActive`).

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `fy_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | ‚Äî | Owning company. **Explicit FK** ‚Üí `companies(company_id)` ON DELETE CASCADE. |
| `start_date` | `DATE` | no | ‚Äî | FY start. Date-only TEXT (`yyyy-mm-dd`) in SQLite ‚Üí `DATE`. |
| `end_date` | `DATE` | no | ‚Äî | FY end. Date-only TEXT ‚Üí `DATE`. Service defaults it to start_date + 1 year ‚àí 1 day. |
| `is_active` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0‚Üífalse, 1‚Üítrue. |
| `is_closed` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0‚Üífalse, 1‚Üítrue. |
| `closing_date` | `DATE` | yes | ‚Äî | Date the FY was closed. Nullable date-only TEXT ‚Üí `DATE`. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT datetime('now')` (app stores ISO strings) ‚Üí `TIMESTAMPTZ`. |

### Constraints / indexes

- **Primary key:** `fy_id`.
- **Unique:** `(company_id, start_date)` ‚Äî mirrors the application-level duplicate guard in `financialYearService.create`.
- **Index:** `idx_financial_years_company_id` on `company_id` (FK column; `getAll`/`setActive` filter by it).

### Relationships

- `financial_years.company_id` ‚Üí `companies.company_id` ‚Äî **EXPLICIT** in the SQLite source (`REFERENCES companies(company_id) ON DELETE CASCADE`). Many financial years belong to one company; deleting a company cascades to its FYs.

No inferred foreign keys for this module (the only `*_id` relationship is the explicit `company_id`).

### Notes on money / quantities

This table has no monetary, quantity, rate, or balance columns, so no `NUMERIC`/floating-point precision concerns apply here.
