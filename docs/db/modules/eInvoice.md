# eInvoice Module — Database Tables

Source: `server/eInvoice/eInvoice.js` (SQLite). Postgres translation in
`docs/db/modules/eInvoice.sql`.

Two tables:
- `einvoice_credentials` — per-company IRP credentials (one row per company)
- `einvoice_records` — generated/cancelled e-Invoice records (IRNs)

General translation notes:
- `INTEGER PRIMARY KEY AUTOINCREMENT` -> `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- SQLite `TEXT DEFAULT (datetime('now'))` columns hold ISO-8601 strings; mapped to
  `TIMESTAMPTZ DEFAULT now()` in Postgres.
- `is_sandbox` is an `INTEGER` 0/1 flag in SQLite -> `BOOLEAN` in Postgres (0 -> false, 1 -> true).
- No money or quantity columns exist in this module, so no NUMERIC mappings are required.

---

## Table: `einvoice_credentials`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `cred_id` | `BIGINT` (identity) | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Explicit FK -> `companies(company_id)` ON DELETE CASCADE. UNIQUE (one credential set per company). |
| `client_id` | `TEXT` | no | — | IRP client id. |
| `client_secret` | `TEXT` | no | — | IRP client secret (sensitive). |
| `username` | `TEXT` | no | — | IRP username. |
| `password` | `TEXT` | no | — | IRP password (sensitive). |
| `app_key` | `TEXT` | no | — | IRP app key. |
| `is_sandbox` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1`. 0 -> false, 1 -> true. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT datetime('now')`; app stores ISO strings. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | Bumped on update via `datetime('now')`. |

**Relationships**
- `company_id` -> `companies(company_id)` — **explicit** in source (`REFERENCES companies(company_id) ON DELETE CASCADE`).
- `UNIQUE (company_id)` added because the service upserts by `company_id` (single credential row per company).

## Table: `einvoice_records`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `irn_id` | `BIGINT` (identity) | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Explicit FK -> `companies(company_id)` ON DELETE CASCADE. |
| `voucher_id` | `BIGINT` | yes | — | **Inferred** FK -> `vouchers(voucher_id)`. Source voucher for the invoice; stored as `voucher_id || null`. |
| `invoice_number` | `TEXT` | no | — | From `invoice_payload.DocDtls.No`. |
| `invoice_date` | `TEXT` | no | — | From `invoice_payload.DocDtls.Dt`. Kept TEXT (IRP `dd/mm/yyyy` format, not ISO) — not mapped to DATE to avoid parse/format loss. |
| `buyer_gstin` | `TEXT` | yes | — | From `invoice_payload.BuyerDtls.Gstin`. |
| `irn` | `TEXT` | yes | — | IRP-assigned Invoice Reference Number. Indexed (lookup/cancel by IRN). |
| `ack_no` | `TEXT` | yes | — | IRP acknowledgement number. |
| `ack_dt` | `TEXT` | yes | — | IRP acknowledgement datetime string (IRP format; TEXT preserved). |
| `signed_invoice` | `TEXT` | yes | — | IRP signed invoice (JWT). |
| `signed_qr_code` | `TEXT` | yes | — | IRP signed QR code (JWT). |
| `ewb_no` | `TEXT` | yes | — | E-way bill number, if generated. |
| `ewb_dt` | `TEXT` | yes | — | E-way bill datetime string (IRP format; TEXT preserved). |
| `status` | `TEXT` | no | `'PENDING'` | `PENDING` (default) \| `GENERATED` (on insert) \| `CANCELLED` (on cancel). |
| `cancel_reason` | `INTEGER` | yes | — | IRP cancellation reason code. Stays INTEGER (a code, not a flag). |
| `cancel_remarks` | `TEXT` | yes | — | Cancellation remarks. |
| `cancelled_at` | `TIMESTAMPTZ` | yes | — | Set via `datetime('now')` on cancel. |
| `raw_response` | `TEXT` | yes | — | JSON string of the full IRP response at generation. Could be JSONB; kept TEXT to match source. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT datetime('now')`; app stores ISO strings. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | Bumped on cancel via `datetime('now')`. |

**Relationships**
- `company_id` -> `companies(company_id)` — **explicit** in source (`REFERENCES companies(company_id) ON DELETE CASCADE`).
- `voucher_id` -> `vouchers(voucher_id)` — **inferred** (no explicit REFERENCES in SQLite; inferred from the `*_id` naming and usage). Modelled `ON DELETE SET NULL` and left nullable.

**Indexes**
- `idx_einvoice_records_company_id` on `(company_id)`.
- `idx_einvoice_records_voucher_id` on `(voucher_id)`.
- `idx_einvoice_records_irn` on `(irn)` — lookups by IRN (`getRecordByIRN`, cancel UPDATE).
- `idx_einvoice_records_company_created` on `(company_id, created_at DESC)` — supports `getRecords` ordering.
