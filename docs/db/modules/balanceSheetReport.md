# balanceSheetReport — Database Schema

Source (SQLite): `server/balanceSheetReport/balanceSheetReport.js`
Postgres DDL: `balanceSheetReport.sql`

This module owns two tables: `balance_sheet_reports` (the report header / saved
configuration) and `balance_sheet_views` (the individual balance-sheet line rows).

General mapping rules applied:
- `INTEGER PRIMARY KEY AUTOINCREMENT` -> `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- `INTEGER` flags with 0/1 / `show_*` / `is_*` semantics -> `BOOLEAN` (0 -> false, 1 -> true).
- `REAL` columns holding money -> `NUMERIC(18,2)`. **Money is never stored as floating point.**
- `TEXT` datetime defaults (`datetime('now')`) -> `TIMESTAMPTZ DEFAULT now()` (app stores ISO strings).
- Date-only `TEXT` (`report_date`, comparison periods) -> `DATE`.
- `NOT NULL` added wherever the SQLite column had `NOT NULL` or a `DEFAULT`.
