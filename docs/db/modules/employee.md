# Employee Module — Database Reference

Source: `server/employee/employee.js`. One table: **`employees`**.

The SQLite schema is created via `CREATE TABLE`, then a set of `ALTER TABLE ADD
COLUMN` statements run idempotently (wrapped in try/catch) to add missing columns
on already-migrated databases, plus a re-add of `employee_category_id`. The
effective column set is captured below.

### Notes
- No money or quantity columns exist in this table, so no `NUMERIC` precision
  decisions apply here.
- `employee_code` uniqueness is enforced only in application logic (a SELECT
  guard), scoped to `company_id` and `is_active = 1`. An optional partial unique
  index is provided (commented out) in `employee.sql`.

## Relationships

All three are **explicit** `REFERENCES` in the SQLite source:

- `employees.company_id` -> `companies.company_id` (ON DELETE CASCADE).
- `employees.employee_category_id` -> `employee_categories.employee_category_id`.
- `employees.employee_group_id` -> `employee_groups.employee_group_id`.

Referenced by (other modules, not defined here): salary structure / payroll
modules reference `employees.employee_id` via `employee_id` columns.
