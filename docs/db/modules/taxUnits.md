# taxUnits Module — Database Schema

Source: `server/taxUnits/taxUnit.js` (SQLite init) -> Postgres DDL in
`docs/db/modules/taxUnits.sql`.

This module owns a single table: **`tax_units`**.

## Table: `tax_units`

Stores excise/tax registration units belonging to a company.

### Constraints & indexes

- **Primary key**: `tax_unit_id`.
- **Unique** (added in Postgres to formalize app logic): partial unique index
  `uq_tax_units_company_active_name` on `(company_id, lower(name)) WHERE is_active = true`.
  Mirrors the create() duplicate check. Not present in the SQLite source.
- **Index**: `idx_tax_units_company_id` on the `company_id` FK column.

### Money / numeric columns

None. This table holds no monetary, quantity, rate, or balance columns, so the
NUMERIC(18,2)/NUMERIC(18,4) money-precision rules do not apply here.

## Relationships

- `tax_units.company_id` -> `companies.company_id`
  **(explicit FK in source)**, `ON DELETE CASCADE`. Deleting a company removes
  its tax units.

No other `*_id` columns exist on this table, so no foreign keys were inferred.
