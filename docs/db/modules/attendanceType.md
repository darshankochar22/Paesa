# attendanceType module — database schema

Source (read-only): `server/attendanceType/attendanceType.js`
Postgres DDL: [`attendanceType.sql`](./attendanceType.sql)

This module owns one table: **`attendance_types`**.

## Table: `attendance_types`

Stores the per-company catalogue of attendance/leave types used by payroll
(Present, Absent, Paid Leave, Overtime, etc.). Soft-deleted via `is_active`.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `attendance_type_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Owning company. **FK (explicit in source)** -> `companies.company_id` `ON DELETE CASCADE`. |
| `name` | `TEXT` | no | — | Type name. Unique per company among active rows (app-enforced; see index). |
| `alias` | `TEXT` | yes | — | Optional short alias. |
| `type` | `TEXT` | no | `'Attendance / Leave with Pay'` | Category label. Seed rows use `'Attendance'` / `'Leave'` / `'Production'`. |
| `unit_id` | `BIGINT` | yes | — | Payroll unit. **FK (explicit in source)** -> `payroll_units.payroll_unit_id`. |
| `period` | `TEXT` | no | `'Per Day'` | Accrual/measurement period. |
| `carry_forward` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0 -> false, 1 -> true. |
| `encashment` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0 -> false, 1 -> true. |
| `max_days` | `NUMERIC(18,4)` | no | `0` | Quantity (max number of days). SQLite `REAL DEFAULT 0`. Fixed-point, **not** floating point. Not money. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1`. Soft-delete flag (delete sets it false). |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. Seeded/system rows; cannot be edited or deleted. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; set on update. |

### Constraints & indexes
- `idx_attendance_types_company_id` — index on `company_id` (FK).
- `idx_attendance_types_unit_id` — index on `unit_id` (FK).
- `uq_attendance_types_company_name_active` — partial **unique** index on
  `(company_id, lower(name)) WHERE is_active = true`. Reflects the service's
  case-insensitive duplicate check on create (no equivalent constraint exists in
  the SQLite source; added in the Postgres translation to enforce the rule).

### Relationships
- `attendance_types.company_id` -> `companies.company_id` (many-to-one, **explicit FK** in source, `ON DELETE CASCADE`).
- `attendance_types.unit_id` -> `payroll_units.payroll_unit_id` (many-to-one, **explicit FK** in source, nullable).

## Notes
- The init function runs a best-effort `ALTER TABLE ... ADD COLUMN <col> TEXT`
  loop over `alias, period, carry_forward, encashment, max_days`. These columns
  already exist in `CREATE TABLE` (and several are not TEXT), so each statement
  fails and is swallowed by `try/catch`. It is a back-fill safeguard for older
  databases and a no-op on a fresh schema. No schema impact in Postgres.
