# DB Module: profitLossReport

Postgres schema contract for the `profitLossReport` backend module. Source of truth:
`server/profitLossReport/profitLossReport.js` (SQLite `init`). DDL: `profitLossReport.sql`.

Two tables:
- **`profit_loss_reports`** — saved P&L report header (one per saved report).
- **`profit_loss_views`** — section/group view rows belonging to a report.

General mappings applied: SQLite `INTEGER PRIMARY KEY AUTOINCREMENT` -> `BIGINT GENERATED BY
DEFAULT AS IDENTITY`; `datetime('now')` TEXT -> `TIMESTAMPTZ DEFAULT now()` (app stores ISO-8601
strings); date-only TEXT -> `DATE`; `0/1` flag integers -> `BOOLEAN` (0=false, 1=true); REAL money
columns -> `NUMERIC(18,2)` (currency — never floating point).

## Notes

- No UNIQUE constraints are defined in the SQLite source for either table.
- Recommended indexes (in `profitLossReport.sql`): `company_id` on `profit_loss_reports`, and
  `report_id` + `company_id` on `profit_loss_views`.
- The `companies` table is owned by a separate module; create it before applying the FK ALTERs.
