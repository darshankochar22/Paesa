## Table: `stock_categories`

Stores hierarchical stock/inventory categories scoped to a company. Rows are
never hard-deleted by the app; `is_active = 0` marks a soft delete.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `sc_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Owning company. **Explicit FK** -> `companies.company_id` (`ON DELETE CASCADE`). |
| `name` | `TEXT` | no | — | Category name. Per-company, case-insensitive uniqueness enforced in app code (among active rows). |
| `alias` | `TEXT` | yes | — | Optional short alias. Added via `ALTER TABLE ... ADD COLUMN` in init (idempotent). |
| `description` | `TEXT` | yes | — | Optional free-text description. |
| `parent_category_id` | `BIGINT` | yes | — | Self-referencing parent. **Explicit FK** -> `stock_categories.sc_id` (`ON DELETE RESTRICT`). |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1` (0/1). 0->`false` (deleted), 1->`true` (active). |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — bumped on update. |

### Type-mapping call-outs
- `is_active` is an INTEGER boolean flag in SQLite; mapped to `BOOLEAN` (0/1 -> false/true).
- `created_at` / `updated_at` are ISO-8601 datetime **strings** in SQLite; mapped to `TIMESTAMPTZ`, datetime defaults translated to `now()`.
- No money or quantity columns exist in this table, so no `NUMERIC` precision concerns apply here.

### Constraints & indexes
- No `UNIQUE` constraint exists in the source; uniqueness of `(company_id, lower(name))` among active rows is enforced in application logic only. A non-unique helper index `idx_stock_categories_company_lower_name` is provided.
- Indexes on both FK columns (`company_id`, `parent_category_id`).

## Relationships
- **`stock_categories.company_id` -> `companies.company_id`** — explicit in source, `ON DELETE CASCADE` (deleting a company removes its categories).
- **`stock_categories.parent_category_id` -> `stock_categories.sc_id`** — explicit in source, self-referencing hierarchy, `ON DELETE RESTRICT` (app also blocks deleting a category that still has active children).
