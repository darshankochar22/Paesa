# stockItem module — database schema

Source of truth: `server/stockItem/stockItem.js` (SQLite `CREATE TABLE` + migrations).
Postgres DDL: `docs/db/modules/stockItem.sql`.

This module owns two tables:
- `stock_items`
- `stock_item_opening_allocations`

## Type-mapping conventions (SQLite -> Postgres)

- `INTEGER PRIMARY KEY AUTOINCREMENT` -> `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- `TEXT` -> `TEXT`.
- `created_at` / `updated_at` (ISO-8601 via `datetime('now')`) -> `TIMESTAMPTZ DEFAULT now()`.
- Date-only text (`mfg_date`, `expiry_date`) -> `DATE`.
- INTEGER used as boolean (`is_*`, `has_*`, `track_*`, `enable_*`) -> `BOOLEAN`; `0`->false, `1`->true.
- REAL holding **quantity / rate (%)** -> `NUMERIC(18,4)`.
- REAL holding **money amounts** -> `NUMERIC(18,2)`. **Money is never floating point.**
- `NOT NULL` added wherever the SQLite column is `NOT NULL` or carries a `DEFAULT`.

## Table: stock_item_opening_allocations

Godown / batch breakdown of an item's opening stock.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| allocation_id | BIGINT IDENTITY | no | identity | Primary key. |
| item_id | BIGINT | no | — | FK -> stock_items.item_id (explicit, ON DELETE CASCADE). |
| godown_id | BIGINT | yes | — | FK -> godowns.godown_id (explicit, ON DELETE SET NULL). |
| batch_number | TEXT | yes | — | |
| mfg_date | DATE | yes | — | Date-only text in SQLite. |
| expiry_date | DATE | yes | — | Date-only text in SQLite. |
| quantity | NUMERIC(18,4) | no | 0 | Quantity. |
| rate | NUMERIC(18,2) | no | 0 | **MONEY** per unit. Never float. |
| amount | NUMERIC(18,2) | no | 0 | **MONEY**. Computed = quantity * rate. |

Note: in SQLite these numeric columns are declared `REAL DEFAULT 0` (no NOT NULL); the
`DEFAULT 0` justifies the `NOT NULL` in Postgres.

### Relationships (stock_item_opening_allocations)

- `item_id` -> `stock_items(item_id)` — **explicit** in source, ON DELETE CASCADE.
- `godown_id` -> `godowns(godown_id)` — **explicit** in source, ON DELETE SET NULL.
