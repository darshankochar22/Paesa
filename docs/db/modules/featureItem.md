# featureItem module — database schema

Source: `server/featureItem/featureItem.js`

This module defines one table: **`feature_items`**. It is seeded at init time
with a fixed set of configuration toggles (`FEATURE_ITEMS` array — accounts,
inventory, gst, online groups) inserted via `INSERT OR IGNORE` joined against
`feature_groups` by `group_key`.

## Table: `feature_items`

| Column | Postgres type | Nullable | Default | Notes |
| --- | --- | --- | --- | --- |
| `feature_item_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `feature_group_id` | `BIGINT` | no | — | FK -> `feature_groups.feature_group_id`. Explicit in source with `ON DELETE CASCADE`. |
| `feature_key` | `TEXT` | no | — | **UNIQUE**. Stable key, e.g. `enable_gst`. |
| `feature_name` | `TEXT` | no | — | Human-readable label. |
| `description` | `TEXT` | yes | — | Optional description. |
| `control_type` | `TEXT` | no | `'boolean'` | UI control type. SQLite default `'boolean'`; NOT NULL added because a DEFAULT is present. |
| `default_value_boolean` | `BOOLEAN` | no | `FALSE` | Stored as `0`/`1` in SQLite (0->false, 1->true). Default 0. |
| `display_order` | `INTEGER` | no | `0` | Ordering within group. |
| `is_mandatory` | `BOOLEAN` | no | `FALSE` | 0/1 boolean flag (0->false, 1->true). Default 0. |
| `is_active` | `BOOLEAN` | no | `TRUE` | Soft-active flag. 0/1 (default 1 -> TRUE). |

### Constraints & indexes

- **Primary key:** `feature_item_id`.
- **Unique:** `feature_key`.
- **Index:** `idx_feature_items_feature_group_id` on `feature_group_id` (FK lookup).
- **NOT NULL** added to every column that has a SQLite DEFAULT or an explicit
  NOT NULL, per the mapping rules.

### Money / quantity columns

None. This table holds no monetary or quantity values, so no `NUMERIC(18,2)` /
`NUMERIC(18,4)` columns are present.

## Relationships

- **`feature_items.feature_group_id` -> `feature_groups.feature_group_id`**
  (many feature items belong to one feature group).
  *Explicit* in the SQLite source:
  `REFERENCES feature_groups(feature_group_id) ON DELETE CASCADE`.
  Emitted as `fk_feature_items_feature_group` in `featureItem.sql`.

No inferred (`*_id` heuristic) foreign keys were needed — the only relationship
is explicitly declared in source.
