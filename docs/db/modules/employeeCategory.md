This module owns one table: **`employee_categories`**.

## Table: `employee_categories`

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `employee_category_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | Primary key. SQLite: `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `company_id` | `BIGINT` | no | — | Owning company. Explicit FK -> `companies(company_id)` `ON DELETE CASCADE`. |
| `name` | `TEXT` | no | — | Category name. Case-insensitive uniqueness enforced in code (not a DB constraint) per active company. |
| `alias` | `TEXT` | yes | — | Optional short name. Defaults to `null` when omitted on create. |
| `allocate_revenue` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0/1 -> false/true. |
| `allocate_non_revenue` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. 0/1 -> false/true. |
| `is_active` | `BOOLEAN` | no | `true` | SQLite `INTEGER DEFAULT 1`. Soft-delete marker (`delete` sets to false). |
| `is_predefined` | `BOOLEAN` | no | `false` | SQLite `INTEGER DEFAULT 0`. Predefined rows cannot be edited or deleted. |
| `created_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))` — ISO-8601 string. |
| `updated_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`. Refreshed on `update`. |

No money/quantity columns in this table — no `NUMERIC` precision concerns here.

### Relationships

- **Outbound (owned, explicit in source):** `company_id` -> `companies(company_id)`, `ON DELETE CASCADE`. Declared explicitly in the SQLite schema; reproduced as `fk_employee_categories_company`.
- **Inbound (referenced by other modules):**
  - `employee_groups.employee_category_id` -> `employee_categories.employee_category_id`
  - `employees.employee_category_id` -> `employee_categories.employee_category_id`

  These inbound references are read by `delete` (to block removal while active groups/employees exist) but their FK constraints belong to the `employeeGroup` / `employee` modules and are not declared in this module's SQL file.

### Indexes

- `idx_employee_categories_company_id` on `(company_id)` — FK lookup.
- `idx_employee_categories_company_lower_name` on `(company_id, lower(name))` — supports the case-insensitive duplicate-name check in `create`.

### Seeding

`seedDefaultEmployeeCategory(company_id)` inserts one predefined row named
`Primary Employee Category` (`is_predefined=1`, `is_active=1`). Not exposed over IPC.
