# DB Module: voucherEntryActions

Source (SQLite): `server/voucherEntryActions/voucherEntryActions.js`
Postgres DDL: `docs/db/modules/voucherEntryActions.sql`

This module owns a single table: **`voucher_entry_actions`**. It logs actions/automation events tied to voucher entry (audit trail, autofill suggestions, mode switches, optional-skip reasons, links to related reports).

## Table: voucher_entry_actions

### Relationships

- **company_id -> companies.company_id** — explicit FK in source, `ON DELETE CASCADE`. Required (NOT NULL).
- **voucher_id -> vouchers.voucher_id** — explicit FK in source, `ON DELETE CASCADE`. Optional.
- **autofill_ledger_id -> ledgers.ledger_id** — explicit FK in source, no cascade rule. Optional.
- **related_report_id (+ related_report_type)** — polymorphic reference. No single FK target; left unconstrained intentionally.

### Type-mapping callouts

- `autofill_amount` holds money. SQLite `REAL` -> Postgres `NUMERIC(18,2)`. Currency is never stored as floating point.
- `is_optional` is a 0/1 flag in SQLite -> Postgres `BOOLEAN` (0=false, 1=true).
- `performed_at` and `created_at` store ISO-8601 datetime strings in SQLite -> `TIMESTAMPTZ`, with the `datetime('now')` default translated to `now()`.
- `action_data` and `additional_details` are JSON-serialized TEXT in SQLite -> `JSONB`. Use `TEXT` instead if you must preserve the service's non-JSON string read-fallback.
