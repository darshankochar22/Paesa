## Table: `tally_features`

SQLite `INTEGER` 0/1 values mapped to Postgres `BOOLEAN` (0 -> `false`, 1 -> `true`).

### Constraints / Indexes
- `PRIMARY KEY (tally_feature_id)`.
- `UNIQUE (company_id)` — **inferred / recommended** (not in SQLite source). All
  queries use `WHERE company_id = ?` and the app keeps exactly one feature row per
  company, so a uniqueness guarantee is appropriate.
- `INDEX idx_tally_features_company_id (company_id)` on the FK column.

### Relationships
- **`company_id` -> `companies(company_id)`** — **explicit** FK in the SQLite source
  (`REFERENCES companies(company_id) ON DELETE CASCADE`). Deleting a company deletes
  its feature row. This is a 1:1 relationship (one feature row per company).

### Money / quantity columns
- None. This table contains no monetary or quantity columns, so no
  `NUMERIC(18,2)` / `NUMERIC(18,4)` precision concerns apply here.
