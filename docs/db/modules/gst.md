# GST Module — Database Schema

Source: `server/gst/gst.js`. Three tables are created by this module, plus two
migration columns added to `stock_groups` (owned by the stock module).

SQLite → Postgres mapping highlights:
- Money columns use `NUMERIC(18,2)`, quantity/rate columns use `NUMERIC(18,4)`. **Never floating point for money or quantities.**
- ISO datetime TEXT (`created_at`, `datetime('now')`) → `TIMESTAMPTZ DEFAULT now()`.
- Date-only TEXT (`effective_from`, `effective_to`, `filed_date`) → `DATE`.
- `is_inter_state` (INTEGER 0/1) → `BOOLEAN` (0 → false, 1 → true).
- `INTEGER PRIMARY KEY AUTOINCREMENT` → `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.

## Table: `gst_hsn_rates`

Company-level, effective-dated HSN/SAC GST rate overrides.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| rate_id | BIGINT IDENTITY | no | identity | PK (was AUTOINCREMENT). |
| company_id | BIGINT | no | — | FK → companies.company_id (explicit, ON DELETE CASCADE). |
| hsn_code | TEXT | no | — | |
| effective_from | DATE | no | — | date-only ISO. |
| effective_to | DATE | yes | — | null = open-ended. |
| taxability | TEXT | no | 'Taxable' | NOT NULL added because source has a DEFAULT. |
| gst_rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| cgst_rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| sgst_rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| igst_rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| cess_rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| type_of_supply | TEXT | no | 'Goods' | 'Goods' \| 'Services'. |
| created_at | TIMESTAMPTZ | no | now() | app stores ISO string. |

**Relationships**
- `company_id` → `companies(company_id)` — **explicit** FK in source, ON DELETE CASCADE.

## Table: `gst_voucher_tax_lines`

Per-voucher GST audit/breakdown lines. Written by `saveVoucherTaxLines`
(gstTaxEngine.js), read during GSTR-1 generation.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| tax_line_id | BIGINT IDENTITY | no | identity | PK. |
| voucher_id | BIGINT | no | — | FK → vouchers.voucher_id (explicit, ON DELETE CASCADE). |
| entry_id | BIGINT | yes | — | FK → voucher_entries.entry_id (explicit, ON DELETE SET NULL). |
| hsn_code | TEXT | yes | — | |
| description | TEXT | yes | — | |
| quantity | NUMERIC(18,4) | no | 0 | qty. |
| unit | TEXT | yes | — | |
| assessable_value | NUMERIC(18,2) | no | 0 | MONEY — exact numeric, never float. |
| tax_type | TEXT | yes | — | 'CGST' \| 'SGST' \| 'IGST' \| 'CESS'. |
| rate | NUMERIC(18,4) | no | 0 | tax rate (percent). |
| amount | NUMERIC(18,2) | no | 0 | MONEY — exact numeric, never float. |
| is_inter_state | BOOLEAN | no | false | SQLite INTEGER 0/1 → false/true. |
| party_gstin | TEXT | yes | — | |
| party_state | TEXT | yes | — | |
| gst_classification_id | BIGINT | yes | — | **inferred** FK → gst_classifications; never written by gst code. |

- `gst_classification_id` → `gst_classifications(gst_classification_id)` — **inferred** from `*_id` naming; not enforced in SQLite source. Verify referenced PK column name before enabling.

Snapshot of a generated GSTR-1 return for a (company, fy, period).

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| export_id | BIGINT IDENTITY | no | identity | PK. |
| company_id | BIGINT | no | — | FK → companies.company_id (explicit, ON DELETE CASCADE). |
| fy_id | BIGINT | no | — | FK → financial_years.fy_id (explicit, ON DELETE CASCADE). |

| filed_date | DATE | yes | — | null until filed. |
| status | TEXT | no | 'Draft' | 'Draft' \| 'Filed'. |
| b2b_json | TEXT | yes | — | JSON string; consider JSONB. |

| hsn_json | TEXT | yes | — | JSON string; consider JSONB. |

## Migration columns on `stock_groups` (foreign table)

`gst.js` also adds two columns to `stock_groups` (owned by the stock module):

| Column | SQLite | Postgres equivalent | Notes |
|---|---|---|---|
| igst_rate | REAL DEFAULT 0 | NUMERIC(18,4) NOT NULL DEFAULT 0 | tax rate; apply in stock_groups module. |
| cess_rate | REAL DEFAULT 0 | NUMERIC(18,4) NOT NULL DEFAULT 0 | tax rate; apply in stock_groups module. |

These are documented for provenance only and are NOT created in `gst.sql`.
