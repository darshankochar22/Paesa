# Banking Module — Database Tables

Source: `server/banking/banking.js` (SQLite). One table: `reconciliations`.

## Table: `reconciliations`

Bank reconciliation records — each row links a voucher entry to its bank-side
matching info, marking it as reconciled.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| `reconciliation_id` | `BIGINT GENERATED BY DEFAULT AS IDENTITY` | no | identity | PK. From SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| `entry_id` | `BIGINT` | no | — | FK -> `voucher_entries.entry_id`. Explicit in source. |
| `voucher_id` | `BIGINT` | no | — | FK -> `vouchers.voucher_id`. Explicit in source. |
| `ledger_id` | `BIGINT` | no | — | FK -> `ledgers.ledger_id` (bank ledger). Explicit in source. |
| `reconciled_date` | `DATE` | yes | — | SQLite TEXT date-only -> DATE. |
| `bank_date` | `DATE` | yes | — | Value date from bank statement. SQLite TEXT date-only -> DATE. |
| `bank_reference` | `TEXT` | yes | — | Bank reference / cheque / UTR number. |
| `reconciled_at` | `TIMESTAMPTZ` | no | `now()` | SQLite `TEXT DEFAULT (datetime('now'))`; app stores ISO-8601 strings -> TIMESTAMPTZ. |

No money or quantity columns exist on this table, so no `NUMERIC` precision
decisions were needed. (General rule for this app: currency -> `NUMERIC(18,2)`,
qty/rate -> `NUMERIC(18,4)`, never floating point.)

## Relationships

All three foreign keys are **explicit** in the SQLite source (declared with
`REFERENCES ... ON DELETE CASCADE`):

- `reconciliations.entry_id` -> `voucher_entries.entry_id` (ON DELETE CASCADE)
- `reconciliations.voucher_id` -> `vouchers.voucher_id` (ON DELETE CASCADE)
- `reconciliations.ledger_id` -> `ledgers.ledger_id` (ON DELETE CASCADE)

The referenced tables (`voucher_entries`, `vouchers`, `ledgers`) belong to other
modules. FK constraints are emitted as separate `ALTER TABLE` statements at the
bottom of `banking.sql` so cross-module table-creation ordering is not an issue.

### Indexes

- `idx_reconciliations_entry_id` on (`entry_id`)
- `idx_reconciliations_voucher_id` on (`voucher_id`)
- `idx_reconciliations_ledger_id` on (`ledger_id`)
