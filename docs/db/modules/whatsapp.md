# WhatsApp Module — Database Schema

Source of truth: `server/whatsapp/whatsapp.js` (SQLite `CREATE TABLE`
statements). Postgres translation in `docs/db/modules/whatsapp.sql`.

Tables: `whatsapp_config`, `whatsapp_templates`, `whatsapp_logs`.

General translation notes:
- `INTEGER PRIMARY KEY AUTOINCREMENT` -> `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- Datetime `TEXT DEFAULT (datetime('now'))` -> `TIMESTAMPTZ DEFAULT now()`. The
  app stores ISO-8601 strings in SQLite.
- `is_active INTEGER DEFAULT 1` is a boolean flag -> `BOOLEAN DEFAULT TRUE`
  (0 -> false, 1 -> true).
- This module has **no money/quantity columns**, so nothing maps to
  `NUMERIC`. (Invoice/statement amounts exist only in transient template
  payloads and are never persisted.)
- `company_id` FKs are **explicit** in the SQLite source
  (`REFERENCES companies(company_id) ON DELETE CASCADE`). The `voucher_id` FK
  is **inferred** (no explicit constraint in source).

---

## whatsapp_config

Stores one WhatsApp Cloud API credential set per company. Upserted by
`company_id` in `whatsappService.saveConfig`.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| config_id | BIGINT IDENTITY | no | identity | PK. SQLite `INTEGER PRIMARY KEY AUTOINCREMENT`. |
| company_id | BIGINT | no | — | FK -> companies (explicit, ON DELETE CASCADE). UNIQUE (one config/company). |
| phone_number_id | TEXT | no | — | Meta phone number id; used in send endpoint path. |
| waba_id | TEXT | no | — | WhatsApp Business Account id. |
| access_token | TEXT | no | — | Meta Graph bearer token. Sensitive. |
| is_active | BOOLEAN | no | TRUE | SQLite `INTEGER DEFAULT 1`; 0/1 -> false/true. `getConfig` filters `is_active = 1`. |
| created_at | TIMESTAMPTZ | no | now() | SQLite `TEXT DEFAULT datetime('now')` (ISO string). |
| updated_at | TIMESTAMPTZ | no | now() | SQLite `TEXT DEFAULT datetime('now')`; set on update. |

**Relationships**
- `company_id` -> `companies(company_id)` — explicit FK in source, `ON DELETE CASCADE`.
- `UNIQUE(company_id)` added (upsert-by-company semantics).

## whatsapp_templates

Catalog of WhatsApp message templates per company.

> **Not used by any IPC channel.** Created by schema init but no SQL in
> `whatsappService.js` reads or writes it. Documented for completeness.

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| template_id | BIGINT IDENTITY | no | identity | PK. |
| company_id | BIGINT | no | — | FK -> companies (explicit, ON DELETE CASCADE). |
| name | TEXT | no | — | Template name. |
| language | TEXT | no | 'en' | SQLite `DEFAULT 'en'`. |
| category | TEXT | yes | — | Optional category. |
| status | TEXT | no | 'PENDING' | SQLite `DEFAULT 'PENDING'`. |
| created_at | TIMESTAMPTZ | no | now() | SQLite `TEXT DEFAULT datetime('now')` (ISO string). |

**Relationships**
- `company_id` -> `companies(company_id)` — explicit FK in source, `ON DELETE CASCADE`.

## whatsapp_logs

Audit log of outbound WhatsApp sends. One row per send attempt (written by
`saveLog`). Read by `getLogs` (`ORDER BY sent_at DESC LIMIT ?`).

| Column | Postgres type | Nullable | Default | Notes |
|---|---|---|---|---|
| log_id | BIGINT IDENTITY | no | identity | PK. |
| company_id | BIGINT | no | — | FK -> companies (explicit, ON DELETE CASCADE). |
| voucher_id | BIGINT | yes | — | Inferred FK -> vouchers. Null for non-invoice sends. |
| to_number | TEXT | no | — | Normalized recipient phone. |
| message_type | TEXT | no | — | INVOICE / PAYMENT_REMINDER / STATEMENT / TEXT. |
| template_name | TEXT | yes | — | Template used (null for free-form text). |
| status | TEXT | no | 'PENDING' | SQLite `DEFAULT 'PENDING'`; runtime SENT / FAILED. |
| wamid | TEXT | yes | — | WhatsApp message id (on success). |
| error | TEXT | yes | — | JSON-stringified Meta error (on failure). |
| sent_at | TIMESTAMPTZ | no | now() | SQLite `TEXT DEFAULT datetime('now')` (ISO string). |

**Relationships**
- `company_id` -> `companies(company_id)` — explicit FK in source, `ON DELETE CASCADE`.
- `voucher_id` -> `vouchers(voucher_id)` — **inferred** FK (no explicit
  constraint in source); modeled `ON DELETE SET NULL` since nullable.

**Indexes**
- `idx_whatsapp_logs_company_id`, `idx_whatsapp_logs_voucher_id`.
- `idx_whatsapp_logs_company_sent_at (company_id, sent_at DESC)` to back the
  `getLogs` query.
